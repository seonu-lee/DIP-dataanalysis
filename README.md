# Soundscape_DIP: LLM 기반 상황 인식 음악 추천 시스템

본 프로젝트는 사용자의 **현재 위치, 활동 목적, 주변 소음(dB)** 등 다차원적인 상황 맥락을 LLM(Large Language Model)이 이해하고, 이에 최적화된 음악 특성을 매칭하여 개인화된 플레이리스트를 제공하는 시스템의 데이터 분석 및 검증 과정을 담고 있습니다.

---

## 프로젝트 구조 및 상세 분석 내용

### 1. EDA & 데이터 전처리 (`/EDA`)

* **환경 소음 데이터**: 한국환경공단 측정망 데이터를 활용하여 5단계 소음 구간(**Noise Band**) 정의.
* *Very Quiet(0-35dB) ~ Very Loud(81dB+)* 구간별 맞춤형 추천 로직 설계.

* **사용자 맥락 데이터 (RAED)**: 위치(Library, Home 등), 행동(Studying, Walking 등) 데이터를 구조화하여 상황별 페르소나 생성.

* **음악 특성 프로파일링**: Spotify API의 정량적 지표(Energy, Valence, Tempo 등)를 활용하여 상황-음악 간 매칭 매트릭스 구축.
* *예: 집중 필요 시 → 낮은 Energy, 높은 Instrumentalness 추천*


### 2.  성능 평가 (`/성능평가(KPI)`)

시스템의 실제 운영 성능을 4가지 핵심 지표로 검증하였습니다.

* **Context-Sentiment Accuracy (81.17%)**: 사용자의 입력 상황(위치, 목표)이 추천 사유에 정확히 반영되었는가.
* **Persona-Genre Accuracy (100.00%)**: 사용자 취향이 페르소나 요약 및 장르 선정에 완벽히 반영됨.
* **Agent Latency**: 평균 **7.62초**(중앙값 6.34초)의 안정적인 응답 속도 확보.
* **Deep Link Conversion (85.15%)**: 추천된 음악에 대한 실제 Spotify 이동 클릭률을 통한 높은 수용도 확인.

### 3. 통계 및 로그 분석 (`/통계분석`)

사전 배포 기간 수집된 실제 사용자 로그 데이터를 기반으로 패턴을 분석했습니다.

* **주요 사용 맥락**: '집', '코워킹 스페이스', '이동 중' 환경에서 '집중'과 '휴식'을 목적으로 하는 사용이 주를 이룸.
* **위치×목표 조합**: 카페-집중, 집-휴식/수면 등 특정 장소와 활동 간의 명확한 결합 패턴 도출.
* **유저 세그먼트**: 리텐션 데이터를 기반으로 유저를 '충성파(Loyal)' 및 '중성파'로 분류하여 타겟팅 개선 방향 수립.

## 기술 스택

* **Language**: Python (Jupyter Notebook)
* **Libraries**: Pandas, NumPy, Matplotlib, Seaborn, Scipy
* **LLM Strategy**: Prompt Engineering & Context Construction
* **Data Sources**: 한국환경공단(소음), Spotify API, RAED Dataset

---

## 실행 가이드

1. **데이터 탐색**: `EDA/` 폴더 내의 노트북을 통해 소음 데이터와 음악 특성이 어떻게 연계되는지 확인하세요.
2. **성능 확인**: `성능평가(KPI)/`에서 실제 모델의 정확도와 사용자 반응 지표를 확인할 수 있습니다.
3. **인사이트 확인**: `통계분석/` 폴더의 결과를 통해 실제 유저들이 어떤 상황에서 음악 추천을 선호했는지 분석할 수 있습니다.

### 기대 효과

이 시스템은 단순한 장르 추천을 넘어 **"현재 카페의 소음이 65dB인 상태에서 집중을 원하는 사용자"**에게 가장 과학적으로 적합한 음악을 추천함으로써, 사용자의 생산성 향상과 심리적 안정에 기여합니다.
